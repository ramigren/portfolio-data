Pipelines Automáticos — Distribución de Fondos

Problema

La consolidación y conciliación de flujos financieros destinados a cuentas institucionales se ejecutaba de manera manual, sin trazas claras de auditoría.

Solución

Creación de un proceso en la nube con tablas de control incrementales y un anti-join lógico riguroso que previene duplicados en los cierres de mes.

Impacto

Reducción total de horas operativas y disponibilidad inmediata de saldos netos listos para la conciliación contable ejecutiva.

🤖 Databricks Pipelineincremental_job.py
import pyspark.sql.functions as F
def run_incremental_clearing(df_raw, df_control):
    df_clean = df_raw.join(df_control, "entidad_id", "left_anti")
    if df_clean.count() > 0:
        return df_clean.withColumn("net_dist", F.col("recaudado") * 0.83)
    return None

Solución

Modelado relacional y limpiezas programadas en SQL para estructurar la base consolidada agregando dimensiones por localidad y categoría tributaria.

Impacto

Establecimiento de la serie histórica oficial utilizada recurrentemente por carteras económicas provinciales para presupuestos anuales.

💾 SQL Relational Querymuc_aggregation.sql
SELECT 
    ano_calendario, localidad_codigo,
    SUM(monto_recaudado) AS total_recaudado,
    COUNT(DISTINCT contribuyente_id) AS padron_activo
FROM db_tributaria.monotributo_unificado
GROUP BY 1, 2 ORDER BY 1 DESC;

from selenium import webdriver
import requests

def alert_telegram(job_title, link):
    token = "TELEGRAM_BOT_TOKEN"
    msg = f"🎯 Nueva Vacante Match: {job_title}\n🔗 Link: {link}"
    url = f"https://api.telegram.org/bot{token}/sendMessage"
    requests.post(url, data={"chat_id": "USER_ID", "text": msg})
